# 15. 공통 컴포넌트

> 근거: [analysis/STEP4_Frontend분석.md](../analysis/STEP4_Frontend분석.md), [STEP5_공통Framework분석.md](../analysis/STEP5_공통Framework분석.md)

## 15-1. Frontend 공통 컴포넌트 목록

| 컴포넌트 | 파일 | 역할 |
|---|---|---|
| 앱 셸 | `templates/base.html` | 3분할(사이드바/탭바/메인/상태바) 레이아웃 골격 |
| 사이드바 트리 | `_sidebar_tree.html` + `sidebar-tree.js` | 재귀 트리 렌더 + 클릭 이벤트 위임 |
| 탭 매니저 | `tab-manager.js` | 탭 생성/전환/닫기, Lazy Loading |
| SplitLayout 엔진 | `split-layout.js` | 레이아웃 트리 재귀 렌더 + 드래그 리사이즈 |
| 검색조건 패널 | `search-panel.js` + `search-fields-config.js` | menuKey별 검색필드 구성 + 접기/펴기 |
| 툴바 | `toolbar.js` | 조회/추가/삭제/저장/인쇄/닫기 버튼(조회만 실배선) |
| 렌더러 레지스트리 | `content-renderers/registry.js` | `registerRenderer`, `renderAgGrid`, `renderKeyValueForm`, `Formatters`, `registerReload` |
| 공통 통신 | `api-fetch.js` | `apiFetch()` — CSRF/에러처리 표준화 |

## 15-2. `renderAgGrid()` — 그리드 공통 컴포넌트

[11장](./11_GridRendering구조.md)에서 상세히 다룸. 한국어 로케일, 공통 기본 옵션(`resizable/sortable/filter`), 툴팁 지연시간 단축이 모든 그리드에 일괄 적용된다.

## 15-3. `renderKeyValueForm()` — 상세정보 표시 공통 컴포넌트

```javascript
function renderKeyValueForm(el, fields) {
    el.classList.add("kv-form");
    fields.forEach(field => {
        // label + readonly input 한 줄씩 렌더
    });
}
```

⚠️ 현재 `input.readOnly = true` 고정 — 표시 전용이다. 실제 수정 가능한 폼(예: `admin.js`의 role 선택 영역)은 이 함수로 일부(읽기전용 필드)만 처리하고, 수정 가능한 필드(select, 버튼)는 별도로 DOM을 직접 구성한다.

## 15-4. `Formatters` — 포맷터 공통 컴포넌트

```javascript
window.Formatters = {
    currency: (v) => v == null || v === "" ? "" : Number(v).toLocaleString(),
    date: (v) => v || "",
};
```

금액 컬럼은 예외 없이 `Formatters.currency`를 `valueFormatter`로 사용한다("원" 접미사 코드는 주석 처리되어 보류 상태).

## 15-5. `DateDefaultRules` — 날짜 기본값 공통 컴포넌트

레지스트리 패턴(`registerRenderer`/`registerReload`와 동일 발상)으로 구현되어 있다.

```javascript
DateDefaultRules.today()          // 오늘 날짜
DateDefaultRules.closingDate()    // 1~10일→10일, 11~20일→11·12일이 월요일이면 10일 아니면 20일,
                                   // 21일~말일→말일, 주말이면 평일로 보정 (Python 원본 로직 JS 포팅)
```

`profit_payment_date`의 검색조건 date 필드가 신규 탭 생성 시 이 규칙으로 기본값을 채운다.

## 15-6. 서버(Backend) 공통 컴포넌트

| 컴포넌트 | 파일 | 역할 |
|---|---|---|
| 로거 | `utils/logger.py` | `setup_logging(app)`, `get_logger(name)` — 전 계층 통합 로깅 |
| 비밀번호 검증 | `auth.py` | `verify_password()` — bcrypt/werkzeug 자동분기 |
| 인증 데코레이터 | `auth.py` | `login_required` — 유일한 공통 인가 장치(⚠️ role 검증 데코레이터는 없음) |
| DB 접속 설정 | `db/settings.py` | `LocalDBSettings`, `ProdDBSettings` |
| DB 커넥션 | `db/local_db.py`, `db/prod_db.py` | 커넥션 풀, `ReadOnlyConnection` |

## 15-7. 공통 컴포넌트 재사용 원칙

신규 화면/도메인을 만들 때는 위 컴포넌트들을 **그대로 재사용**하고 새로 만들지 않는 것이 원칙이다. 특히:

- 그리드는 `renderAgGrid()`, 통신은 `apiFetch()`, 로깅은 `get_logger(__name__)`을 예외 없이 사용한다.
- 엔진 성격의 컴포넌트(SplitLayout, TabManager, registry.js)는 수정하지 않고, 설정(SCREEN_LAYOUTS 등)과 신규 렌더러 파일만 추가한다.
