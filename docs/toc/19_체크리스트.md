# 19. 체크리스트

## 19-1. 신규 메뉴 개발 체크리스트 (요약 — 상세는 [17장](./17_신규메뉴개발절차.md))

새 화면 1개 추가 시 아래 8개 파일을 확인/작업한다.

- [ ] `menu_data.py` — 트리 leaf 노드 추가
- [ ] `static/js/screen-layouts.js` — `SCREEN_LAYOUTS`에 레이아웃(유형 A/B/C) 추가
- [ ] `static/js/content-renderers/{도메인}.js` — `registerRenderer()` 추가 (신규 도메인이면 파일 신규 생성 + `base.html`에 `<script>` 태그 추가, 접근 빈도가 낮으면 `tab-manager.js`의 `DOMAIN_SCRIPT_MAP`에 등록해 Lazy Loading 대상으로)
- [ ] `static/js/search-fields-config.js` — `SEARCH_FIELDS_CONFIG` 추가 (선택, 없으면 기본값 폴백)
- [ ] `repository/{도메인}_repository.py` — 신규 (로컬DB는 `get_local_db_connection()`, 상용DB는 `get_prod_db_connection()`만 사용)
- [ ] `service/{도메인}_service.py` — 신규 (routes는 이 계층만 호출, 검증/날짜계산 등 비즈니스 로직 배치)
- [ ] `routes/{도메인}_routes.py` + `app.py`에 Blueprint 등록 — **Blueprint 등록 누락이 가장 흔한 실수**
- [ ] `{도메인}-api.js` + `base.html`에 `<script>` 태그 — `apiFetch()` 기반으로 작성

**절대 수정하지 않는 엔진 파일**: `sidebar-tree.js`, `tab-manager.js`, `split-layout.js`, `search-panel.js`, `content-renderers/registry.js`. 이 파일들을 고쳐야 할 필요가 생겼다면, 설계 원칙에서 벗어난 요구사항일 가능성이 높으므로 먼저 재검토한다.
