# 관리자(설정) 화면 구현 계획 — 개요

작성일: 2026-07-11
관련 기능: 사이드바 하단 "설정" 진입 → 관리자 화면(사용자 목록/상세/권한)

## 1. 배경
- 기존 뼈대(사이드바 트리메뉴, 탭 관리, 툴바, 검색패널, SplitLayout, AG Grid)는 구현 완료 상태.
- `routes/admin_routes.py`에 `admin_bp`는 등록만 되어 있고 실제 라우트는 없음.
- 현재 `users` 테이블 컬럼: `id, email, password, role` 4개뿐 (확장 예정 상태).

## 2. 결정된 범위 (2026-07-11 논의 기준)
| 항목 | 결정 |
|---|---|
| 권한 관리 방식 | 우선 Role 기반(admin/user)으로 뼈대만. 추후 메뉴별 세부 권한 매트릭스(RBAC)로 확장 가능하게 구조만 분리 |
| 이번 작업 범위 | 현재 `users` 테이블과 **실제 연동**까지 구현. 테이블 확장(컬럼 추가/신규 테이블)은 **설계안만 제안**, 구현은 다음 단계 |
| 사이드바 진입 방식 | 기존 방식대로 "설정" **탭**으로 열기 (새 라우트 페이지/모달 아님) |

## 3. 화면 컨셉
- 사이드바 하단 우측에 별도 클래스로 설정(⚙) 버튼 배치
- 클릭 시 "설정" 탭 오픈 (기존 트리메뉴 leaf 클릭 → 탭 오픈 패턴 재사용)
- 탭 내부는 좌/우 분할:
  - 좌측: 사용자 목록 (AG Grid) — 이메일/role 등 기본 정보
  - 우측: 탭 구조(`기본정보` / `권한정보`) — 좌측에서 선택한 사용자의 상세 표시 및 role 변경
  - 좌측 클릭 → 우측 fetch 연동 (`GET /admin/users/<id>`)

## 5. 진행 순서
1단계(사이드바 버튼) → 2단계(백엔드) → 3단계(프론트) → 4단계는 문서만 작성(구현하지 않음)

**진행 상태 (2026-07-11 기준): 1~4단계 전부 완료.**
- 1~3단계: 실제 코드 구현 및 검증 완료 (사이드바 버튼 → API → SPA 화면 전체 동작 확인됨)
- 4단계: 스키마 확장 설계안 문서화 완료 (`04_schema_proposal.md`) — **구현은 별도 요청 시 진행**
